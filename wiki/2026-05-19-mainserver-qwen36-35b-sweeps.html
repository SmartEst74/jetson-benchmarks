Title
Mainserver Qwen3.6 35B optimization sweeps
Date
2026-05-19
Tried
Removed the Twingate connector from mainserver, upgraded llama-cpp-turboquant to upstream 2b61ea24e, installed NCCL, rebuilt, then ran sweeps over split mode, fit margins, context size, thread settings, no-mmproj text-only mode, and speculative draft decoding.
Method
Single-request benchmark against the local OpenAI-compatible endpoint using a deterministic prompt: Output the numbers 1 to 120 separated by commas and spaces only. Measured with timings.predicted_per_second from the server response.
Experiment
  • Baseline multimodal service after connector removal and tuning
  • Auto-fit vs forced -ngl 999
  • layer vs row split modes
  • Context sweeps: 131k, 65k, 32k, 16k
  • Speculative ngram modes
  • Draft model tests using Qwen3.5-0.8B-Q3_K_S.gguf
  • Text-only mode (no mmproj) + speculative draft parameter tuning
Conclusion
Breakthrough achieved. The best stable sustained result so far is ~8.34 tok/s using the same main model in a text-only profile with a small Qwen-family draft model. The best safe multimodal baseline is about ~6.4 tok/s.

Current understanding

The main blockers to 10 tok/s sustained are the host hardware constraints: a 2-core Pentium G4400 and two GTX 1050 Ti 4 GB GPUs. The large gain came from shifting the work into a text-only speculative profile rather than continuing to micro-tune the multimodal baseline.
