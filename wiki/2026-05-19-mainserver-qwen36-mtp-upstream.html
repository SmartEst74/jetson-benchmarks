Best launch command

/home/hal/src/llama.cpp/build-mtp/bin/llama-server \
  -m /home/hal/models/Qwen3.6-35B-A3B-MTP-GGUF/Qwen3.6-35B-A3B-UD-Q8_K_XL.gguf \
  -dev CUDA0,CUDA1 \
  -ts 4,1 \
  -c 32768 \
  -b 512 -ub 64 \
  -fa on \
  -t 2 -tb 2 \
  --threads-http 1 \
  -np 1 \
  -ctk q8_0 -ctv q8_0 \
  --spec-type draft-mtp \
  --spec-draft-n-max 4 \
  --n-cpu-moe 39 \
  --op-offload \
  --no-ui \
  --no-ui-mcp-proxy \
  --no-slots \
  --host 0.0.0.0 --port 8080

Important trade-off

This MTP profile is now the default on :8080 and preserves the exact Q8_K_XL MTP model quality. Full GPU MoE placement is still not possible on this 2x4GB system with the exact Q8 model, but the tuned 4,1 tensor split shifts more work to GPU, keeps KV on GPU, and materially reduces the earlier CPU-heavy bias.
