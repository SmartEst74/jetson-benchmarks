Agent Roles & Model Mapping

Different tasks need different strengths. We map each agent role to the best model that actually runs on Jetson — backed by benchmarks, not vibes.

Hot-Swap Architecture

The Jetson runs one model at a time (8GB). The hot-swap API automatically loads the optimal model for each agent role — trading a ~10s model switch for dramatically better task performance.

POST http://jetson:8001/api/switch
{"role": "tool_caller"}

→ Stops current model
→ Loads Nanbeige4-3B-Thinking Q8_0 (best BFCL: 51.4)
→ Returns {"model": "Nanbeige4-3B-Thinking", "api": "http://jetson:8000/v1", "ready": true}

See api/hot-swap.py for the implementation and scripts/bench-role.sh to run role-specific benchmarks.
