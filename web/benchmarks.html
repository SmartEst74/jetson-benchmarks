Methodology

Measured performance: median of 3 runs using scripts/bench-chat.sh with a standardized coding prompt (512 max tokens, temperature 0.6). Timings from llama.cpp timings JSON. MAXN_SUPER mode, jetson_clocks active, cache dropped before each run.

Predicted tok/s: (68 GB/s / GGUF_size_GB) * 0.69 — derived from 69% bandwidth efficiency measured on hardware.

BFCL v4 scores: from Berkeley Function Calling Leaderboard (commit f7cf735, 2025-12-16). Qwen3.5-4B (49.7) from model card.

Tiers: T1 ≤ 4.5GB (headroom) · T2 = 4.5-6GB (fits) · T3 = 6-7GB (swap risk) · T4 = >7GB or rejected.
