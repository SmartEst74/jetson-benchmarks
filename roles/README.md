# Agent Role Definitions

These role definitions are copied directly from [agency-agents](https://github.com/msitarzewski/agency-agents) by [@msitarzewski](https://github.com/msitarzewski) under the MIT license.

Each role defines a specialized AI agent persona with identity, mission, critical rules, deliverables, and workflow. We benchmark LLMs on the Jetson Orin Nano Super by testing how well each model performs when acting as these real-world agent roles.

## Roles Used for Benchmarking

| Role | Source Division | Benchmark Focus |
|------|----------------|-----------------|
| [Frontend Developer](engineering-frontend-developer.md) | Engineering | React/Vue component generation, CSS, accessibility |
| [Backend Architect](engineering-backend-architect.md) | Engineering | System design, database schemas, API architecture |
| [Code Reviewer](engineering-code-reviewer.md) | Engineering | Bug detection, security review, actionable feedback |
| [Security Engineer](engineering-security-engineer.md) | Engineering | Threat modeling, OWASP analysis, secure code |
| [Technical Writer](engineering-technical-writer.md) | Engineering | README quality, API docs, tutorials |
| [AI Engineer](engineering-ai-engineer.md) | Engineering | ML pipeline design, model deployment, MLOps |
| [Performance Benchmarker](testing-performance-benchmarker.md) | Testing | Load testing, optimization, metrics analysis |
| [API Tester](testing-api-tester.md) | Testing | Endpoint validation, security testing, test automation |
